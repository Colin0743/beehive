# 管理系统需求文档

## 概述

蜂巢平台需要一个完整的管理系统，用于管理网站的项目、用户账号等各类内容。管理系统应包含内容管理、用户管理、数据统计等核心功能。

## 功能需求

### 1. 管理员角色和权限

#### 1.1 管理员角色
- **超级管理员（Super Admin）**：拥有所有权限
- **内容管理员（Content Admin）**：可以管理项目和内容
- **用户管理员（User Admin）**：可以管理用户账号

#### 1.2 权限控制
- 只有管理员可以访问管理系统
- 不同角色有不同的权限范围
- 普通用户无法访问管理后台

#### 1.3 管理员账号
- 支持通过特殊邮箱或配置创建管理员账号
- 管理员账号在用户注册时不可见
- 管理员可以提升其他用户为管理员

### 2. 内容管理（项目管理）

#### 2.1 项目列表管理
- 查看所有项目列表
- 支持搜索和筛选（按状态、分类、创建时间等）
- 显示项目详细信息（标题、创建者、状态、参与人数等）
- 支持批量操作（批量删除、批量修改状态）

#### 2.2 项目编辑
- 编辑项目基本信息（标题、描述、分类等）
- 修改项目状态（激活、暂停、完成）
- 删除项目
- 查看项目详情和日志

#### 2.3 项目审核
- 审核新创建的项目
- 标记违规项目
- 隐藏/显示项目

### 3. 用户管理

#### 3.1 用户列表
- 查看所有注册用户
- 支持搜索和筛选（按邮箱、注册时间等）
- 显示用户详细信息（姓名、邮箱、注册时间、项目数等）
- 显示用户状态（正常、禁用、删除）

#### 3.2 用户操作
- 查看用户详情
- 编辑用户信息
- 禁用/启用用户账号
- 删除用户账号
- 重置用户密码
- 提升用户为管理员
- 查看用户创建的项目列表

#### 3.3 用户统计
- 总用户数
- 活跃用户数
- 新注册用户数（按时间段）

### 4. 数据统计

#### 4.1 平台概览
- 总项目数
- 活跃项目数
- 总用户数
- 活跃用户数
- 总参与人数
- 平台总时长

#### 4.2 项目统计
- 按分类统计项目数
- 按状态统计项目数
- 项目创建趋势（按时间）
- 最受欢迎的项目（按参与人数）
- 项目完成率

#### 4.3 用户统计
- 用户注册趋势（按时间）
- 用户活跃度统计
- 用户参与项目统计
- 用户角色分布

#### 4.4 数据可视化
- 使用图表展示统计数据
- 支持时间范围筛选
- 导出统计数据（CSV格式）

### 5. 系统设置

#### 5.1 平台设置
- 平台名称和描述
- 平台Logo和主题色
- 系统公告

#### 5.2 分类管理
- 添加/编辑/删除项目分类
- 分类排序

## 技术实现

### 1. 路由结构
```
/admin
  /dashboard          # 数据统计仪表盘
  /projects           # 项目管理
  /users              # 用户管理
  /settings           # 系统设置
```

### 2. 权限检查
- 在管理路由中添加权限检查中间件
- 非管理员用户重定向到首页
- 显示友好的错误提示

### 3. 数据存储
- 管理员角色信息存储在User类型中
- 使用localStorage存储（MVP阶段）
- 后续可迁移到数据库

### 4. UI设计
- 使用侧边栏导航
- 响应式设计，支持移动端
- 使用表格展示列表数据
- 使用图表展示统计数据

## 验收标准

### 功能验收
- ✅ 管理员可以登录管理系统
- ✅ 普通用户无法访问管理系统
- ✅ 可以查看和管理所有项目
- ✅ 可以查看和管理所有用户
- ✅ 可以查看数据统计
- ✅ 权限控制正常工作

### 性能验收
- ✅ 页面加载时间 < 2秒
- ✅ 列表数据加载流畅
- ✅ 搜索和筛选响应迅速

### 安全验收
- ✅ 只有管理员可以访问管理功能
- ✅ 敏感操作需要确认
- ✅ 操作日志记录（可选）
