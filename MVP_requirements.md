# "蜂巢" AI视频协作平台 MVP需求文档

## 1. 项目概述

"蜂巢"是一个参考 [Kickstarter](https://www.kickstarter.com/) 模式的AI视频线上协作网站。平台的核心目标不是筹集资金，而是**"筹集"AI视频的制作时长与创意**。项目发起人（导演）提出创意蓝图，吸引工蜂（算力提供者）加入，共同完成一部AI视频作品。

本项目旨在快速搭建一个最小可行产品（MVP），验证核心协作流程。因此，在保证核心需求达成的前提下，我们将尽可能复用或模仿Kickstarter的现有成熟架构和页面布局，仅对文字、图标和核心逻辑进行“蜂巢”概念的替换。

## 2. 核心概念替换

为了快速开发，我们将Kickstarter的核心概念进行如下替换：

| Kickstarter 概念 | “蜂巢”平台概念 | 说明 |
| :--- | :--- | :--- |
| **资金 (Funding)** | **视频时长 (Video Duration)** | 项目的目标不再是金额，而是最终AI视频的分钟数。 |
| **筹款目标 (Funding Goal)** | **时长目标 (Duration Goal)** | 发起人设定一个希望完成的视频总时长，如“5分钟”。 |
| **筹款进度条** | **时长进度条** | 实时显示当前已完成的视频时长占总目标的百分比。 |
| **支持者 (Backer)** | **工蜂 / 参与者** | 用户以工蜂身份参与项目。 |
| **回报 (Reward Tiers)** | **参与身份 (Participation Role)** | 用户支持项目不再是为了获得实体或虚拟回报，而是以工蜂身份参与项目。 |
| **项目更新 (Updates)** | **项目日志 (Project Log)** | 发起人定期发布项目进展，展示新生成的镜头、美术设定等。 |

## 3. 用户角色与职责

平台采用二元结构，主要有两种核心用户角色：

### 3.1. 项目发起人 (Initiator)

- **职责**：项目的"导演"和"总负责人"。
- **核心操作**：
    1.  **创建项目**：提供项目demo视频、剧本简介、人物/场景设定、美术风格参考。
    2.  **设定目标**：定义最终的视频【时长目标】。
    3.  **建立工作流**：提供标准的、流程化的提示词（Prompt）生成步骤，指导工蜂工作。
    4.  **招募工蜂**：吸引工蜂加入项目。
    5.  **管理与剪辑**：在Telegram群组中分配任务给工蜂，接收工蜂生成的视频片段，并负责最终的视频剪辑。
    6.  **更新进度**：在项目管理页面更新【时长进度】，发布【项目日志】。
    7.  **建立沟通渠道**：创建并提供Telegram等外部沟通群组链接。

### 3.2. 工蜂 (Worker Bee)

- **职责**：项目的"算力支持者"，在站外（如Telegram）进行实际的视频生成工作。
- **核心操作**：
    1.  **加入项目**：以"工蜂"身份在网站上加入项目，表明参与意愿。
    2.  **接收任务**：在项目发起人建立的Telegram群组中，接收发起人提供的提示词和任务要求。
    3.  **生成视频**：使用自己的算力（或工具）根据任务要求生成AI视频片段。
    4.  **交付成果**：将生成的视频片段在Telegram上返还给发起人。

## 4. MVP核心功能模块

### 4.1. 用户认证系统
- [ ] 用户注册
- [ ] 用户登录
- [ ] 个人信息编辑（头像、简介）

### 4.2. 项目创建与编辑
- [ ] 提供一个表单页面，让【发起人】可以创建项目。
- [ ] 表单需包含以下字段：
    - 项目标题
    - 项目封面图/视频
    - 项目简介（支持Markdown）
    - **【时长目标】**（如：300秒）
    - Demo视频链接
    - 剧本、人设、场景设定等详细描述
    - **Telegram群组链接**
    - 所属分类

### 4.3. 项目浏览与发现
- [ ] **首页**：展示精选的、热门的、最新的项目。
- [ ] **项目列表页**：可以按分类筛选项目。
- [ ] **项目卡片 (Project Card)**：在列表页展示项目缩略信息，包含封面、标题、发起人、当前时长进度。

### 4.4. 项目详情页
- [ ] 展示项目的全部信息（创建时填写的内容）。
- [ ] 醒目地展示**【时长进度条】**。
- [ ] 清晰地展示发起人和工蜂的列表。
- [ ] 提供明确的行动号召（Call to Action）按钮：
    - **"加入项目"**：用户点击后跳转到项目的Telegram群组链接，并以【工蜂】身份加入项目。
    - 若项目未设置Telegram群组链接，则提示"该项目暂无群组链接"。
- [ ] **【项目日志】**区域：展示发起人发布的所有更新。

### 4.5. 项目管理仪表盘 (Dashboard)
- [ ] **仅项目发起人可见**。
- [ ] **核心功能：更新项目进度**。
    - 提供一个输入框，让发起人可以手动更新“当前已完成时长”。例如，从 `90/300秒` 更新到 `120/300秒`。
    - 提交后，项目详情页的进度条应实时或准实时更新。
- [ ] **发布【项目日志】**：一个简单的表单，用于发布图文更新。
- [ ] **成员管理**：查看和管理已加入的工蜂。

### 4.6. 个人中心 (Profile)
- [ ] 展示用户的基本信息。
- [ ] 分页或列表展示用户**发起**的项目、**参与**的项目。

## 5. 简化与排除功能 (Out of MVP Scope)

为了确保快速上线，以下功能在MVP阶段**不予考虑**：

- **站内聊天/私信功能**：所有沟通强制引导至发起人提供的外部群组（如Telegram）。
- **资金交易**：平台不涉及任何金钱交易、支付、提现等功能。
- **自动算力点数系统**：工蜂提供的“点数”在MVP阶段是一个荣誉或身份标识，不涉及真实的技术对接和点数增减。
- **复杂的权限管理**：MVP阶段只区分“发起人”和“其他用户”两个基本权限等级。
- **算法推荐**：首页的项目推荐可采用手动或按最新、最热等简单规则排序。
